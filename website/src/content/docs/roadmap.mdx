## Next up

**Scrolling polish** — the layout engine exists and tiles windows in a
horizontal strip, but it's a minimal first pass: several scrolling-specific
actions (center column, cycle width, consume/expel) ship unbound, and the
viewport/column behavior needs tightening before it feels finished.

## Known limitations

These are current constraints, not future work items:

- **Per-desktop sizing** — layout choice per (output, desktop) exists in the
  KCM; master ratio, count, and scrolling column width override per monitor
  only, not per virtual desktop
- **Divider drag with gaps** — the ratio you get from dragging the master/stack
  divider is approximate when gaps are non-zero
- **Per-app rules are partial** — always-tile and per-app layout assignment
  exist in `TilingRules` but aren't fully exposed in the KCM yet
