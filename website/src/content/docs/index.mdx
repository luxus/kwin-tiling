Dynamic tiling for KDE Plasma — master-stack, stacked, scrolling, and centred
layouts, adjustable gaps, float/ignore rules, and a settings panel — built
**into** stock KWin, not as a separate compositor.

## What is this?

KWin Tiling adds automatic window tiling to the compositor you already run. New
windows snap into a layout; you move, resize, and float them with keyboard
shortcuts and the mouse. Everything is configured through *System Settings* and
persists across restarts.

The tiling logic started in
[theblackdon/kineticwe](https://gitlab.com/theblackdon/kineticwe) (the KineticWE
fork). This project takes **only** that feature and ships it as a small patch
on top of upstream KWin.

## Patch, don't fork

Instead of maintaining a renamed compositor fork, we patch stock `kdePackages.kwin`:

- **New code** lives as normal source files under `pkgs/kwin-tiling/src/` and is
  copied into the KWin tree at build time.
- **Existing KWin files** are changed through a small `hooks.patch` (~750 lines)
  — CMake wiring, input hooks, workspace/window integration.

That keeps the compositor binary, protocol support, multi-monitor handling, and
Plasma integration identical to upstream. When KDE fixes a bug in KWin, you get
it automatically. On a nixpkgs bump you only re-test the patch surface, not an
entire fork.

## What we dropped from KineticWE

We cherry-picked the tiling feature and left the rest behind:

- Binary rename and full KineticWE compositor rebrand
- Distro install scripts and custom build hacks
- Hand-rolled window borders and built-in rounded corners

For rounded corners and focus outlines on frameless windows, use the separate
[`kde-rounded-corners`](https://github.com/matinlotfali/KDE-Rounded-Corners)
effect instead.

## What we shipped

Beyond what KineticWE had, we completed several pieces (master count, mouse
resize inside columns, live KCM changes, cross-monitor robustness) and packaged
it cleanly for NixOS.

At a glance:

- **Four layout modes** — master-stack, stacked, scrolling, and centred
- **Gaps** — space between tiles and around the screen edge, per-monitor overrides
- **Window rules** — float or ignore apps by class/title; dialogs auto-float
- **Full control** — keyboard shortcuts, drag-to-swap, divider resize, settings KCM
- **Nix packaging** — flake module and overlay; no compositor fork to maintain

See [Features](features) for the full breakdown of each layout, rule, and
improvement.

## Get started

Add the flake module to a NixOS host that should run tiling:

```nix
imports = [ inputs.kwin-tiling.nixosModules.kwin-tiling ];
```

Enable tiling in `~/.config/kwinrc` (`[Tiling] Enabled=true`) or in *System
Settings → Window Management → Tiling*. Shortcuts and daily use are covered on
the [Usage & Shortcuts](usage) page.

For packaging details and source layout, see `pkgs/kwin-tiling/README.md` in the
repository.
